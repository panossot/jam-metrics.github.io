Wildfly Optional Subsystem

JAM-Metrics are available as a Wildfly Optional Subsystem. This library can be used to facilitate the creation, record and monitoring of metrics for any deployment on Wildfly Server. This is done by annotating any class or method with the annotations @Metric and @DBStore. An example of usage can be found under the folder AutomatedMetricsApiTest.

Before you deploy the generated .war file on Wildfly server, the JAM-Metrics Extension needs to be loaded as a Subsystem on the Wildfly Server.

How to install the JAM-Metrics Wildfly extension :

  1. Go to the directory WildflyAddExtensionSubsystem/AutomatedMetricsSubsystemExtension.
  2. Execute the maven command : mvn install -DHome=The home directory of your Wildfly installation.
  3. Restart Wildfly server.

After loading the JAM-Metrics Subsystem on Wildfly, deploy the .war file on the server to see the metrics being printed.
